Quicksort

Quicksort is a divide-and-conquer algorithm that runs in O(n log n) time on average, and O(n2) time in the worst case. The basic structure of the algorithm is as follows:

QUICKSORT(A):
If A has fewer than 2 elements:
Return A
Pick a pivot p from A
L ← QUICKSORT(Things in A ≤ p)
U ← QUICKSORT(Things in A > p)
Return L + [p] + U

This algorithm is good for sorting both fixed-size contiguous arrays, as well as arrays built on linked lists (if you are smart about it). It is is also easy to understand, and is generally pretty fast. These all make it an attractive choice for many applications.

In the following demo, the two solid black bars represent the current region we are partitioning. The orange marked bar represents the pivot value, and the dotted blue lines mark the unpartitioned region. This is a direct translation of the C algorithm in the appendix.

Merge Sort

Merge sort is also a divide-and-conquer algorithm that runs in O(n log n) time. The algorithm is structured as follows:

MERGE-SORT(A):
If A has fewer than 2 elements:
Return A
L ← MERGE-SORT(First half of A)
U ← MERGE-SORT(Second half of A)
Return MERGE(L, U)

The MERGE operation is not shown here, but it merges two sorted subarrays into a single sorted array. The code for MERGE depends on the exact array implementation used. If arrays are required to be contiguous in memory, then a temporary array to contain the merged array is required. (There is a variant of merge sort that makes in-place merging possible, but it is complex and is not included here). If linked lists are being used, then the MERGE operation is quite simple and requires no extra memory. Merge sort is a very popular linked list sorting algorithm for this reason.

For merge sort, and merge sort only, an additional operation "shift" has been added, which moves a bar down the array and shifts everything in between up one spot. This operation is central to the merge operation. Merge sort is much more appropriate for linked lists, where merging can be done very quickly and without using any extra memory.

The black bars represent the region that is currently being merged. The blue dashed line marks the boundary between the left and right sorted subarrays. The right subarray is merged into the left one.

Insertion Sort

Insertion sort is a very basic sorting algorithm that closely mimics how humans sort things. The typical case has a running time of O(n2). The basic structure of the algorithm is as follows:

INSERTION-SORT(A):
S ← []
For each e in A:
Insert e into S so that it is sorted
Return S
